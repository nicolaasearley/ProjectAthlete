## Prerequisites

- **Docker** >= 20.10
- **Docker Compose** >= 2.0 (or docker-compose >= 1.29)
- **Git** (for cloning repository)
- **4GB RAM minimum** (8GB recommended)
- **10GB disk space** minimum

### Installing Prerequisites

#### Docker Installation

**Linux (Ubuntu/Debian):**
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
```

**macOS:**
Download and install from [Docker Desktop](https://www.docker.com/products/docker-desktop)

**Verify Installation:**
```bash
docker --version
docker compose version  # or docker-compose --version
```

## Quick Start

The fastest way to deploy:

```bash
# 1. Clone repository to server
git clone https://github.com/nicolaasearley/ProjectAthlete.git
cd ProjectAthlete

# 2. Make scripts executable (if needed)
chmod +x scripts/*.sh deploy.sh

# 3. Run setup script (creates .env, generates secrets, creates directories)
./scripts/setup.sh

# 4. Edit .env file with your configuration
nano .env

# 5. Deploy application
./deploy.sh
```

That's it! Your application is now running.

**For updates later:**
```bash
git pull
./deploy.sh --build  # Rebuild images with new code
```

### Repository Information

- **GitHub Repository:** https://github.com/nicolaasearley/ProjectAthlete.git
- **Clone Command:** `git clone https://github.com/nicolaasearley/ProjectAthlete.git`

**Security Note:** Sensitive files (`.env`, `data/uploads/`, `data/backups/`) are excluded from Git via `.gitignore`. Only code and configuration templates are tracked in the repository. Never commit sensitive credentials.

## Detailed Setup

### Step 1: Clone Repository to Server

```bash
# SSH into your server
ssh admin@your-server-ip

# Create project directory (if needed)
sudo mkdir -p /opt
sudo chown $USER:$USER /opt

# Clone the repository
cd /opt
git clone https://github.com/nicolaasearley/ProjectAthlete.git
cd ProjectAthlete
```

**Alternative:** If you prefer a different location:
```bash
git clone https://github.com/nicolaasearley/ProjectAthlete.git ~/ProjectAthlete
cd ~/ProjectAthlete
```

### Step 2: Initial Setup

Run the setup script to:
- Create `.env` file from template
- Generate JWT secrets automatically
- Create required directories

```bash
./scripts/setup.sh
```

### Step 3: Configure Environment

Edit the `.env` file with your settings:

```bash
nano .env
```

**Required Settings:**
- `POSTGRES_PASSWORD` - Strong password for database
- `JWT_ACCESS_SECRET` - Already generated by setup script
- `JWT_REFRESH_SECRET` - Already generated by setup script
- `FRONTEND_URL` - Your frontend URL (for CORS)

**Docker vs PM2:**

For Docker deployment, use service names:
- `DATABASE_URL=postgresql://postgres:password@postgres:5432/fitness_earley?schema=public`
- `REDIS_URL=redis://redis:6379`
- `API_URL=http://backend:3000` (internal)

For external access, configure ports in `.env`:
- `API_PORT=3000` (exposed to host)
- `FRONTEND_PORT=5173` (exposed to host)

### Step 4: Deploy

```bash
./deploy.sh
```

This will:
1. Build Docker images (if needed)
2. Start database and cache services
3. Run database migrations
4. Start all application services
5. Perform health checks

### Step 5: Verify Deployment

Check service status:
```bash
docker-compose ps
```

Or run health check script:
```bash
./scripts/health-check.sh
```

Access the application:
- Frontend: http://localhost:5173 (or your server IP)
- Backend API: http://localhost:3000/api/v1

## Environment Variables

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `POSTGRES_PASSWORD` | Database password | `StrongPassword123!` |
| `JWT_ACCESS_SECRET` | JWT access token secret | Auto-generated |
| `JWT_REFRESH_SECRET` | JWT refresh token secret | Auto-generated |
| `DATABASE_URL` | Full database connection string | See below |
| `REDIS_URL` | Redis connection string | See below |

### Database Configuration

**For Docker (recommended):**
```env
DATABASE_URL=postgresql://postgres:password@postgres:5432/fitness_earley?schema=public
REDIS_URL=redis://redis:6379
```

**For External Database:**
```env
DATABASE_URL=postgresql://user:pass@db.example.com:5432/fitness_earley?schema=public
REDIS_URL=redis://redis.example.com:6379
```

### Application Configuration

```env
# Ports (exposed to host)
API_PORT=3000
FRONTEND_PORT=5173

# Frontend URL (for CORS)
FRONTEND_URL=http://localhost:5173,http://your-domain.com

# Storage
UPLOADS_DIR=/data/uploads
MEDIA_PUBLIC_URL=/media
```

### Optional Variables

- `SMTP_*` - Email configuration
- `GOOGLE_CLIENT_ID/SECRET` - OAuth (future)
- `DOMAIN` - Domain name for SSL
- `LETSENCRYPT_EMAIL` - Let's Encrypt email

See `env.example` for all available options.

## Deployment Workflow

### Initial Deployment

```bash
# Clone repository
git clone https://github.com/nicolaasearley/ProjectAthlete.git
cd ProjectAthlete

# One-time setup
./scripts/setup.sh

# Configure environment
nano .env

# Deploy application
./deploy.sh
```

### Updating Application

When you push changes to GitHub, update your deployment:

```bash
# Pull latest code from repository
git pull

# Rebuild images and redeploy
./deploy.sh --build

# Or just restart services (if no code changes)
docker-compose restart
```

**Important:** Always backup before updating:
```bash
./scripts/backup-db.sh
```

### Stopping Services

```bash
docker-compose down
```

To also remove volumes (WARNING: deletes data):
```bash
docker-compose down -v
```

### Viewing Logs

```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f postgres
```

## Database Management

### Running Migrations

```bash
./scripts/migrate-db.sh
```

Or manually:
```bash
docker exec project-athlete-backend npx prisma migrate deploy
```

### Creating Backup

```bash
./scripts/backup-db.sh
```

Backups are stored in `data/backups/` with timestamp.
Old backups are automatically cleaned (default: 7 days retention).

### Restoring Backup

```bash
# List available backups
ls -lh data/backups/

# Restore specific backup
./scripts/restore-db.sh data/backups/backup_20240101_120000.sql.gz
```

### Manual Database Access

```bash
# Connect to PostgreSQL
docker exec -it project-athlete-postgres psql -U postgres -d fitness_earley

# Run SQL query
docker exec project-athlete-postgres psql -U postgres -d fitness_earley -c "SELECT COUNT(*) FROM users;"
```

### Prisma Studio (Database GUI)

```bash
docker exec -it project-athlete-backend npx prisma studio
```

Then access at http://localhost:5555

## Troubleshooting

### Services Won't Start

1. **Check logs:**
   ```bash
   docker-compose logs
   ```

2. **Check container status:**
   ```bash
   docker-compose ps
   ```

3. **Check health:**
   ```bash
   ./scripts/health-check.sh
   ```

### Database Connection Issues

1. **Verify database is running:**
   ```bash
   docker ps | grep postgres
   ```

2. **Check connection:**
   ```bash
   docker exec project-athlete-postgres pg_isready -U postgres
   ```

3. **Verify DATABASE_URL in .env:**
   - For Docker: Use `postgres` as hostname
   - Check password matches

### Port Already in Use

If ports 3000 or 5173 are already in use:

1. **Change ports in .env:**
   ```env
   API_PORT=3001
   FRONTEND_PORT=5174
   ```

2. **Or find and stop conflicting service:**
   ```bash
   lsof -ti:3000 | xargs kill -9
   ```

### Out of Disk Space

```bash
# Check disk usage
df -h

# Clean up Docker
docker system prune -a

# Remove old backups
find data/backups -name "*.sql.gz" -mtime +30 -delete
```

### Container Keeps Restarting

```bash
# Check logs for errors
docker-compose logs backend

# Check resource limits
docker stats

# Restart specific service
docker-compose restart backend
```

#### Backend Container Cannot Find dist/main.js

If you see `Error: Cannot find module '/app/packages/backend/dist/main.js'`, the build output is missing. This is usually caused by a stale Docker build cache. Rebuild the backend without cache:

```bash
# Rebuild backend image without cache
docker-compose build --no-cache backend

# Then restart services
docker-compose up -d
```

Or rebuild all services:

```bash
# Rebuild all images without cache
docker-compose build --no-cache

# Then restart
docker-compose up -d
```

## Moving to New Server

### Step 1: Backup Current Deployment

```bash
# Backup database
./scripts/backup-db.sh

# Backup uploads
tar -czf uploads-backup.tar.gz data/uploads/

# Backup .env file (contains your configuration)
cp .env .env.backup
```

### Step 2: Setup on New Server

```bash
# SSH into new server
ssh admin@new-server-ip

# Clone repository
cd /opt
git clone https://github.com/nicolaasearley/ProjectAthlete.git
cd ProjectAthlete

# Transfer data files from old server
# (You'll need to do this from your local machine or old server)
```

### Step 3: Transfer Data Files

From your local machine or old server, copy the data:

```bash
# Transfer database backup
scp user@old-server:/opt/ProjectAthlete/data/backups/backup_*.sql.gz user@new-server:/opt/ProjectAthlete/data/backups/

# Transfer uploads
scp user@old-server:/opt/ProjectAthlete/uploads-backup.tar.gz user@new-server:/opt/ProjectAthlete/

# Transfer .env file
scp user@old-server:/opt/ProjectAthlete/.env.backup user@new-server:/opt/ProjectAthlete/.env
```

### Step 4: Restore on New Server

```bash
cd /opt/ProjectAthlete

# Extract uploads
tar -xzf uploads-backup.tar.gz

# Deploy application
./deploy.sh

# Restore database
./scripts/restore-db.sh data/backups/backup_*.sql.gz
```

**Note:** Since code is in Git, you only need to transfer:
- Database backups
- Uploaded media files
- `.env` configuration file

### Step 4: Verify

```bash
./scripts/health-check.sh
```

## Production Considerations

### Security

1. **Use strong passwords** in `.env`
2. **Keep JWT secrets secure** - never commit to git
3. **Use reverse proxy** (Nginx/Traefik) for SSL
4. **Limit exposed ports** - only expose frontend port
5. **Regular backups** - automate with cron

### Performance

1. **Resource limits** - add to docker-compose.yml:
   ```yaml
   deploy:
     resources:
       limits:
         cpus: '2'
         memory: 2G
   ```

2. **Database optimization** - adjust PostgreSQL settings
3. **Caching** - ensure Redis is properly configured
4. **CDN** - serve static assets via CDN

### Monitoring

```bash
# View resource usage
docker stats

# Check service health
./scripts/health-check.sh

# Monitor logs
docker-compose logs -f
```

### Automation

Create a cron job for backups:

```bash
# Add to crontab: crontab -e
0 2 * * * cd /opt/ProjectAthlete && ./scripts/backup-db.sh
```
